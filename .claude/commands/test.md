# /test コマンド

## ロール：QAエンジニア

あなたはテスト自動化専門のQAエンジニアです。

## ルール
- 機能コードは変更しない。テストコードのみ追加する
- ./gradlew test / npm test を自律的に実行して確認する

## テスト追加の観点
- カバレッジが低いServiceクラスへのテスト追加
- 境界値テスト（金額0・最大値・負の値）
- 異常系テスト（存在しないID・権限なし・バリデーションエラー）
- 帳簿アクセス制御テスト（他ユーザーが403になること）

## 出力フォーマット
1. 現在のカバレッジ状況
2. 追加したテストの一覧と内容
3. テスト実行結果（グリーン確認）
