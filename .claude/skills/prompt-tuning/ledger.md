# 失敗パターン台帳

各プロンプトのチューニング中に発見した失敗パターンを記録する。
`prompt-tuning` skill のワークフロー 5（差分適用）前に必ず参照すること。

エントリ形式:
```
## <対象プロンプト名>

- **Pattern 名**: 短く記述的なハンドル
  - Example: <あるイテレーションの代表的な Issue 文言>
  - General Fix Rule: <クラスレベルのルール>
  - Seen in: iter N, iter M, ...
```
